    • Stack Memory: Used for static memory allocation, such as function calls and local variables. It follows a LIFO structure and is faster. Heap Memory: Used for dynamic memory allocation, such as objects and data at runtime. It is slower but provides flexibility.
    • Compiled Language: Translates the entire code into machine code before execution (e.g., C, C++). Faster execution but slower compilation., Interpreted Language: Translates code line-by-line during runtime (e.g., Python, JavaScript). Easier debugging but slower execution.
    • Inheritance: Reuse properties/methods of a parent class in a child class., Polymorphism: One function can behave differently based on input (method overriding/overloading)., Encapsulation: Wrapping data and methods into a single unit (class) and restricting access using access modifiers., Abstraction: Hiding implementation details and exposing only necessary functionality.
    • Binary Tree: A tree where each node has at most two children (left and right)., Binary Search Tree (BST): A binary tree where the left child contains values less than the parent, and the right child contains values greater than the parent.
    • Normalization: The process of organizing data to reduce redundancy and improve integrity., Importance: It eliminates duplicate data, ensures data dependencies are logical, and improves database performance.
    • Process: Independent execution unit with its own memory space., Thread: Subset of a process sharing the same memory but executing independently.
    • Deadlock: A situation where processes are waiting for each other indefinitely, causing a halt., Prevention: Avoid circular waits, allocate resources carefully, and use resource ordering.
    • OSI Model: A conceptual framework with 7 layers (Physical, Data Link, Network, Transport, Session, Presentation, Application) for standardizing network communication. .
    • Atomicity: Transactions are all or nothing., Consistency: Ensures data integrity., Isolation: Transactions are independent., Durability: Changes are permanent after a commit.
    • QuickSort: Divide-and-conquer; pivot-based partitioning. Average: 𝑂 ( 𝑛 log ⁡ 𝑛 ) O(nlogn), Worst: 𝑂 ( 𝑛 2 ) O(n 2 )., MergeSort: Divide-and-conquer; merges sorted halves. Always 𝑂 ( 𝑛 log ⁡ 𝑛 ) O(nlogn). Use Case: QuickSort is faster for small datasets; MergeSort is stable and preferred for linked lists.
    • SQL: Relational; structured data; uses tables (e.g., MySQL)., NoSQL: Non-relational; flexible schema; suited for unstructured data (e.g., MongoDB).
    • A memory management technique using disk space to simulate additional RAM. Allows running large programs on limited physical memory.

    • Microservices: Small, independent services that communicate via APIs., Monolithic: Single application with tightly coupled components., Difference: Microservices offer better scalability and fault isolation.
    • Version Control: Tracks code changes, facilitates collaboration., Git: Distributed, fast, and integrates with platforms like GitHub.
    • Uses Garbage Collection to manage memory. Divided into heap and stack memory.
    • Interpreted, dynamically typed, easy syntax, extensive libraries, and support for OOP and functional programming.
    • Binary Tree: Each node has ≤ 2 children., AVL Tree: Self-balancing binary search tree., B-Tree: Multi-way tree optimized for disk storage.
    • Tree: Hierarchical structure, no cycles, one root., Graph: General structure, may have cycles, multiple connections.
    • Implemented using a heap. Min-heap for lowest priority first; max-heap for highest priority first.
    • Dynamic Programming: Solves subproblems, stores results for reuse., Greedy: Makes optimal local choices for a global solution.
    • Efficient string matching using a partial match table to avoid redundant comparisons.
    • A sequence of operations treated as a single unit. Managed using ACID properties (Atomicity, Consistency, Isolation, Durability).
    • Clustered: Data stored in index order., Non-clustered: Separate structure, contains pointers to data.
    • INNER JOIN: Common data in both tables., LEFT JOIN: All from left table, matching right table., RIGHT JOIN: All from right table, matching left table., FULL JOIN: Combines LEFT and RIGHT.
    • Multitasking: Running multiple processes., Multithreading: Multiple threads in a single process., Multiprocessing: Multiple processes in separate memory spaces..
    • Paging: Fixed-size memory blocks., Segmentation: Variable-sized memory blocks (logical division).
    • Used in process synchronization to control resource access using counters.
    • Preemptive: Processes can be interrupted (e.g., Round Robin)., Non-preemptive: Runs to completion (e.g., FCFS).
    • Hub: Broadcasts data to all devices., Switch: Directs data to specific devices., Router: Connects networks
    • Security system that monitors and controls network traffic based on rules.
    • CI (Continuous Integration): Automates code integration/testing., CD (Continuous Delivery/Deployment): Automates code release to production.
